The proposal in detail

The current page already earns trust well (Trustpilot, 97% / 16–25% / 125,000+). The single biggest mobile friction is that the ask, the location picker and the form, sits below a full-screen hero and a VSL video, so a visitor has to scroll through two screens before they can act. Everything below is aimed at shortening the path from intent to registration on mobile, where the majority of traffic lands.

What changes, and why

ChangeWhyMetric it moves
Pull the location picker and a short form (first name, email, phone) above the foldThe ask currently sits behind the hero and the VSL; on mobile that is two scrolls before anyone can registerRegistration CVR up, CPA down
Tighten the headline from five-plus lines to one benefit lineLong serif headline fills the first screen and slows comprehension on a phoneCVR (faster to the offer)
Condense proof to one scannable line, keep the Trustpilot barKeeps the credibility that is working without pushing the form further downCVR (trust retained, less scroll)
Turn the plain location list into tappable cards, nearest date first, with seat scarcityEasier one-thumb selection and a reason to act now instead of laterCVR + registration intent
Keep the VSL, but place it below the formVisitors who want the video still get it; it no longer gates the registration actionCVR without losing the sales narrative

How we would test it (single variable at a time)

  1. Round 1 Form + location above the fold vs current. This is the biggest lever, so it runs first.
  2. Round 2 Tightened headline vs current headline.
  3. Round 3 Seat scarcity and nearest-date-first ordering vs neutral list.
  4. Each round is an A/B test in a single region so the read is clean, before rolling the winner across markets.

How we measure it

  • Primary: registration conversion rate (visits to registration) and cost per registration.
  • Guardrail: show rate, so we never trade registration volume for attendance quality.
  • Read: per region, once each variant clears a minimum volume.

What stays

  • The Trustpilot bar and the 97% / 16–25% / 125,000+ proof, which already work.
  • The VSL video and the overall brand look.
  • The ugly landing page and the AI flyer stay off the table.

How it feeds marketing

  • A higher on-page conversion rate lowers cost per registration, which lets budget scale further inside the same event cap.
  • It pairs with the add-to-calendar signal test: a cleaner form flow plus optimizing toward higher-intent registrants compounds on show rate and cost per attendee.
  • The client can implement page changes quickly, so we can move from mockup to a live single-region test fast.
